# ゲーム仕様

## 概要

ターン制RPG REST API。キャラクターを作成し、ダンジョンを探索して敵と戦うゲームバックエンド。

## 機能一覧

| 機能 | 内容 |
|------|------|
| キャラクター作成 | 名前・職業選択・初期ステータス反映 |
| ステータス | HP・MP・攻撃力・防御力・素早さ・レベル・経験値・スキルポイント |
| ダンジョン探索 | 部屋の移動・イベント発生 |
| ターン制戦闘 | 攻撃・スキル・アイテム・逃げる |
| アイテム・装備 | インベントリ・装備による能力変化 |
| スキル・魔法 | 職業ごとのスキル習得・スキルポイント消費 |
| ショップ | ダンジョン外でアイテム購入・売却 |
| セーブ・ロード | DB永続化（セッションもDB保存） |

## キャラクター仕様

- 認証・アカウント管理は本スコープ外とする
- 職業はDBマスタで管理（データ追加で拡張可能）
- キャラクター作成時、選択した職業の初期ステータス（base_hp / base_mp / base_attack / base_defense / base_speed）を反映する
- キャラクター作成時、初期職業を `character_jobs` に記録する
- キャラクター作成時の初期値は `level = 1`、`exp = 0`、`gold = 0`、`skill_points = 0` とする
- レベルアップ時にステータスが自動上昇する（上昇量は職業ごとに異なる）
- ステータスには素早さ（speed）を含める

## 職業仕様

- 職業ごとに初期ステータスが異なる
- 職業ごとに習得できるスキルが異なる（レベル条件あり）
- スキル習得にはスキルポイントを消費する
- レベルアップ時にスキルポイントを 1 付与する
- スキルごとに習得コストを設定できる
- 職業・スキルの種類はDBマスタで管理

## ダンジョン仕様

- 複数のダンジョンが存在し、推奨レベルが設定されている
- 固定マップ（roomsテーブルで管理）
- 部屋タイプ：normal / boss / treasure / rest
- 最終部屋はボス固定
- 探索中・戦闘中の状態はDBに保存（ブラウザを閉じても再開可能）
- normal部屋では 1〜3 体の敵とエンカウントする
- boss部屋ではボス 1 体 + 取り巻き 0〜2 体でエンカウントする
- rest部屋は移動直後に HP / MP を最大値の 50% 回復する（最大値は超えない）
- rest部屋 / treasure部屋は 1 回のみイベント発生とし、再訪問時は効果を発生させない
- treasure部屋ではアイテム 1〜2 個 + ゴールドを獲得する

## 戦闘仕様

- 部屋移動時に自動で戦闘開始（ドラクエ方式）
- アクション：attack / skill / item / run
- 攻撃・スキル・アイテム使用時は対象敵を指定できる
- 行動順は speed を比較し、キャラクターの speed が敵以上なら先攻、それ以外は後攻とする
- 戦闘は複数敵に対応し、全敵撃破で勝利とする
- 戦闘勝利時：経験値・ゴールド・アイテムドロップ
- レベルアップ判定は戦闘結果レスポンスに含める
- 戦闘結果レスポンスにはターンごとの戦闘ログを含める
- 死亡時：`explore_session` を削除・`characters` の hp を 1 に更新・所持金を半減
- buff系スキルの効果は戦闘中にターン管理する

## 装備仕様

| 部位 | 説明 |
|------|------|
| weapon | 武器 |
| armor | 防具 |
| helmet | 兜 |
| shield | 盾 |
| accessory | アクセサリ |

## ショップ仕様

- ダンジョン外に設置
- ゴールドで購入
- アイテムごとに売却可否と売却価格を設定できる
- 売却専用アイテムは `price = 0`、`sell_price` のみを持つ

## レベルアップ仕様

- EXP閾値はDBテーブル（`level_exp_thresholds`）で管理
- レベルアップ時にステータスが自動上昇（上昇量は職業ごとに異なる）
- レベルアップ時にスキルポイントを 1 付与する
- レベルアップ情報は戦闘結果レスポンスに含める

## 転職仕様

- 職業にはランクがある（初級職 / 中級職 / 上級職 / 最上級職）
- 転職するには前提職業をすべてマスターしている必要がある
- マスターとは、その職業で `master_level` に到達することを指す
- 複数の職業の掛け合わせで上位ランクの職業に転職できる
- 転職するとレベルは 1 にリセットされる
- 転職・習熟状況は `character_jobs` で管理する

## 探索イベント仕様

### rest

- 部屋に移動した瞬間に自動発動する
- HP / MP をそれぞれ最大値の 50% 回復する
- 回復量は整数で管理し、最大値を超える回復は行わない
- `visited_rooms` で訪問済みを管理し、同一探索中は 1 回のみ有効とする
- レスポンスには `event = "rest"` と回復量を含める

### treasure

- 部屋に移動した瞬間に自動発動する
- アイテム 1〜2 個 + ゴールドをランダムで獲得する
- 宝箱の中身は `room_treasures` で定義する
- `visited_rooms` で訪問済みを管理し、同一探索中は 1 回のみ有効とする
- レスポンスには `event = "treasure"` と獲得報酬を含める

## 追加で記載・検討する機能

- **バフ / デバフ管理**: buff系スキルの効果量と残りターンを `battle_effects` で管理する
- **敵の行動パターン**: 敵も通常攻撃以外にスキルを使用できるよう `enemy_skills` を定義する
- **戦闘ログ**: 各ターンの行動と結果をレスポンスに含め、クライアントでそのまま表示できる形にする
